How to build cloud solutions

A solution for a cloud application does not solely consist of how the application is architected. It is important to define how the system interacts with other considerations. Many of these other elements are essential to maintain control and compliance with a regulated arena. I have broken this down into 6 sections.

Application Lifecycle Management: How we build, test and release the software/hardware into the cloud

Security: How we secure data and access to cloud resources

Operations: How we know what our application and infrastructure is doing at all times

Data: How we store and use data correctly in a cloud environment

Application Styles: How an application is designed to utilise the benefits of the cloud

Integration: How the application consumes and is consumed by other services in the cloud.
