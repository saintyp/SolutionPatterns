Operations

OSomeone somewhere will have to monitor or troubleshoot issues within the solution that you design. This will fall in to the operations or run elements of the application. You need to consider that everything that happens with in the solution needs to be logged into some sort of system so that monitoring and alerting can be used.

Logging

you need to consider logging from all sources including but not limited to. Network flow logs, IAM logs, Application monitoring, infrastructure metrics, api access.

Monitoring

Once you have collected the data what metrics do you want to look for and what do you want them to give you.

Alerting

What alerts are useful and who should they go to. Are some alerts set to automatically do certain actions (like auto scaling)

What does good look like for operations?

Any design that allows you to understand what is happening in the underlying infrastructure and application and allows you to act on this.
