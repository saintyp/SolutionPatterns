Background/Intro

 

For all servers within the cloud their audit point for compliance will be the CI/CD pipeline.  For images this will be the creation of a gold image which has been tested for vulnerabilities.

 

This gold image is then consumed and the application is deployed to the image.  All testing and compliance is then completed.

 

There currently three stages/pipelines in use:  Image creation, Development & Test, Production

Pattern 1: Deployment guidelines.

 

Each element in the pipeline is controlled by one or more toolsets. Links to the various deployment guidelines are below

 

·        Pipelines can be created in jenkins.

·        Image creation is via packer

·        Automation of code deploy is controlled via ansible

·        Testing/capacity & availability is supplied via the testing pattern

·        Vulnerability testing is via the vulnerability testing pattern

·        Compliance requirements for the image are supplied in the image security reqirements

·        Environment build is controlled via terraform
