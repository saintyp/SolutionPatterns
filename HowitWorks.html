How it Works Part 1

A cloud solution is the combination of an application design and its integration with the eco system around it (i.e logging or data compliance). How the application integrates with the rest of your business is as important as how the application functions.

Use the reference architectures on the site to identify an good application architecture for your product. That you can then customize to your needs.

From the reference architectures it will list the mandatory integration patterns that you will need to include in the final solution.

The integration patterns themselves are not use case specific so each pattern includes an implementation guide so that you can understand what elements you will need to specify in the final design.

Each reference architecture and implementation guide will include terraform templates so that you can quickly try your solution with our test environments.

Once you have created your Cloud solution submit it back to the community so the next person can benefit from your knowledge.
