Cloud Architecture Framework

Introduction

The cloud architecture framework describes how to approach bringing a product to market on the cloud. This framework contains several elements that have different responsibilities but ultimately it is there to produce a well architected cloud applications that can integrate with the companies eco system for creating, securing, validating and operating a product.

The digram below shows the flow through the framework and it components.

Design Principles

A set of overarching principles that all cloud designs should seek to follow and implement.

Application Styles

A collection of types of application designs that are suited to cloud deployment. Including but not limited to Microservices, event driven, data analytics.

Data Styles

A collection of different ways in which data can be consumed within cloud providers. Including but not limited to Persistant data, Transient data, Analytics data, Backup & restore of data.

Application design

An overall solution that performs the neccessary functions to provide the product functionality. The application will still need to fit within the overall eco system that provides patterns for creating, securing, validating and operating the application.

Integration Requirements

In order to function within the company the application needs to fit within an ecosystem. The main components are functions that allow:

  • Creation of environments and deployment of application
  • Securing product service and environment
  • Validation of product environment settings
  • Operation of product environment and service

Format of the Generic Patterns

The generic pattern is there to provide a high level view of the architecture. It will have five sections:

  • Introduction
  • Simplified pattern diagram with explanation
  • Deployment guidelines
  • Owner
  • Status (In Draft, Published, awaiting approval, Approved)

Deployment guidelines

These contain information for the solution engineer or implementation user that allows them to consume the eco system service in practical terms.

Detailed Pattern or generic solution

This contains the detail of the architectural design for the eco system service owner about how they will provide the service. This may be cloud service provider specific (as it with IAM pattern for aws) or more generic (SMTP solution for all cloud providers).
