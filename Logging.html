Logging Pattern

Background

The logging pattern defines the architecture of the logging solutions that are approved for use in. The logging pattern also defines where to send your log files from both an operational and a application support view. The pattern defines that all logs for operations will be sent to the cloud providers logging mechanism(Cloudwatch). The logs are then collected via a connector to Qradar which stores the logs in the on premise Qradar.

Deployment Guide

This deployment guide defines defines the needs of the end consumer of the pattern (i.e qradar is the consumer of the logging pattern and it would require certain information for it to make sense). The logging deployment guide takes the requirements of the pattern (For which the required infrastructure components are already in place) and turns them into a practical requirements for your solution. If will help you define and include all the required elements for your application solution. For example:

Tag all instances with, BusinessName,Role/function

Instance name, Instance Role, Instance application group.
