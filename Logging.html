Logging

Logging Pattern

The logging pattern defines where to send your log files from both an operational and a application support view. The pattern defines that all logs for operations will be sent to the cloud providers logging mechanism(Cloudwatch). The logs are then collected via a connector to Qradar which stores the logs in the on premise Qradar.

Logging Implementation Guide

The logging implementation guide takes the requirements of the pattern (For which the required infrastructure components are already in place) and turns them into a practical implementation for you solution. If will help you define and include all the required elements for your application solution. For example:

Tag all instances with, BusinessName,Role/function

Minimum tags required for qradar:
Instance name, Instance Role, Instance application group.

Instance metrics (Core): Tagged as above
CPU Usage, Memory Usage, Disk Queue length
